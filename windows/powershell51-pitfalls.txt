====================================================================
 Windows PowerShell 5.1 の落とし穴メモ
====================================================================

Windows 標準の PowerShell(5.1)は新しい PowerShell 7 と挙動が
違う点が多い。実際に踏んだものを中心に記録。

--------------------------------------------------------------------
1. 【実際に踏んだ】BOM なし UTF-8 の .ps1 は日本語が化けて構文エラー
--------------------------------------------------------------------

  症状:
    日本語を含む .ps1 を実行すると
    「The string is missing the terminator」等の謎のパースエラー。
    エラーメッセージ内の日本語が「繝薙Ν繝・」のように化けている

  原因:
    PowerShell 5.1 は BOM なしのファイルをシステム既定エンコーディング
    (日本語環境では Shift-JIS)として読む。
    VS Code 等は既定で「BOM なし UTF-8」で保存するので化ける

  対策(どれか):
    a) スクリプト内の文字列・コメントを ASCII(英数字)のみにする
    b) BOM 付き UTF-8 で保存する
       (VS Code: 右下のエンコーディング表示 → "UTF-8 with BOM で保存")
    c) PowerShell 7 (pwsh) を使う(BOM なし UTF-8 が既定)

--------------------------------------------------------------------
2. && と || が使えない
--------------------------------------------------------------------

  cmd A && cmd B   → 5.1 ではパースエラー(7 では使える)

  代替:
    A; if ($?) { B }     # A 成功時のみ B
    A; B                 # 無条件に続けて実行

--------------------------------------------------------------------
3. Out-File / リダイレクトの既定が UTF-16
--------------------------------------------------------------------

  "text" > file.txt や Out-File の既定エンコーディングは UTF-16 LE。
  他のツール(git, javac 等)が読むファイルは明示指定する:

    Set-Content file.txt -Encoding utf8 -Value "..."
    Out-File file.txt -Encoding utf8

--------------------------------------------------------------------
4. その他 5.1 に「ない」もの
--------------------------------------------------------------------

  ・三項演算子 a ? b : c        → if/else で書く
  ・null 合体 ??, ?.            → if ($null -eq $x) で書く
  ・ConvertFrom-Json -AsHashtable → 5.1 は PSCustomObject のみ

--------------------------------------------------------------------
5. スクリプト実行がブロックされたとき
--------------------------------------------------------------------

  「このシステムではスクリプトの実行が無効になっています」
  → 実行ポリシーの制限。一時的に回避するには:

    powershell -ExecutionPolicy Bypass -File .\script.ps1

  恒久的に緩めるなら(自分の環境のみ・要検討):
    Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
